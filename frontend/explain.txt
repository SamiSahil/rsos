Why this order is best
Because if you start directly with a random module like billing.js or syncCenter.js, you may get confused:

where the data comes from
who calls the functions
where the UI is rendered
which state variables are shared
how the module is loaded
But if we start from the top:

HTML shows the app skeleton
CSS shows visual structure
main.js shows where JS begins
app.js shows app orchestration
config/store show shared foundations
modules become much easier after that
